\chapter{System Overview}

This chapter presents the PawPal system architecture, data design, domain models, and detailed design diagrams. The system employs a modular monolithic backend, cross-platform mobile frontend, and specialized AI services.

\section{Architectural Design}

The system uses a three-tier architecture: presentation tier, business logic tier, and data tier. AI services run as isolated Python microservices for efficient ML framework integration.

\subsection{High-Level Architecture}

The system architecture can be visualized as a client-server model with the following major components:

\begin{figure}[H]
    \centering
    \includegraphics[width=1.0\textwidth]{ThesisFigs/system_overview_box_line.png}
    \caption{PawPal System Overview - Box and Line Diagram}
    \label{fig:system_overview}
\end{figure}

The architecture consists of Flutter mobile clients, Golang API gateway with eight business logic modules, specialized AI services (FastAPI chatbot, CNN model), PostgreSQL/ Redis/ ChromaDB data tier, and external integrations (Groq LLM, OpenStreetMap, Firebase, blockchain).

\subsection{Modular Monolithic Architecture}

All eight modules deploy as a single unit with clear boundaries and interfaces. Benefits include direct function calls for performance, simplified deployment, centralized logging, atomic transactions, and lower infrastructure costs. Each module uses consistent layers: Handlers (HTTP), Services (business logic), Models (entities), Repositories (database access), and Utils (shared functions).

\subsection{Layered Architecture}

The system implements five-tier separation of concerns from presentation (Flutter UI) through API gateway, business logic, data access, to storage layers.

\section{Data Design}

The data layer uses PostgreSQL (primary relational database), Redis (caching/sessions), and ChromaDB (vector embeddings). Schema design follows normalization principles with PostgreSQL JSONB for flexible metadata.

\subsection{Data Storage Strategies}

\textbf{PostgreSQL:} Primary database storing user accounts, pet profiles, health records, marketplace data, and transactions with structured relationships.

\textbf{Redis Caching:} Temporary storage for active user sessions, frequently accessed pet data, and API responses to improve system performance.

\textbf{ChromaDB:} Vector database containing veterinary documents enabling intelligent chatbot responses through semantic search.
